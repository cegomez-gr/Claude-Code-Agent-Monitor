# Prompt 03: Generate PR-by-PR Plan

Read the architecture docs and generated specification.

Task:

Generate a PR-by-PR implementation plan.

Rules:

- Each PR must be small.
- Each PR must have acceptance criteria.
- Each PR must be independently reviewable.
- Avoid mixing refactor and feature work.
- Start by preserving existing behavior.

Output:

- `docs/generated/pr-plan.md`
